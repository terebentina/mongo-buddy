## Plan Mode

- Make the plan extremely concise. Sacrifice grammar for the sake of concision.
- Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one.
- If a question can be answered by exploring the codebase, explore the codebase instead.
- For each question, use the question tool and provide your recommended answer.
- After the plan is finalized, before asking the user if they are ready to code, run a review of the plan with fresh eyes. This will help broaden your understanding of the plan and codebase and help you catch any issues you might have missed previously.
- The plan should include unit tests and building each feature in TDD style: write the test first and run it - it should fail, write the implementation code and run the test again - it should pass.
