## Codebase Patterns
- Use `pnpm` not `npm` for all package operations
- Pre-existing lint errors (4 errors, 1 warning) exist in ConnectionDialog.tsx and button.tsx — don't try to fix these as part of migration
- Use `useRender` from `@base-ui/react/use-render` for render prop pattern — handles ref merging and prop merging correctly
- Import path pattern: `@base-ui/react/<feature-name>` (e.g., `@base-ui/react/use-render`, `@base-ui/react/collapsible`, `@base-ui/react/dialog`)
- Base UI `render` prop pattern: `<Trigger render={<CustomElement>children</CustomElement>} />` — children go inside the render element
- Base UI animation data attrs: `data-[open]` and `data-[closed]` (not `data-[state=open]`/`data-[state=closed]`)
- When Radix props (e.g., `onInteractOutside`) don't exist in Base UI, destructure & discard them in the wrapper component
- Base UI Popover requires `Positioner` wrapper between Portal and Popup (unlike Radix which puts positioning on Content directly)
- Base UI Popover.Positioner uses `align` prop (not `alignment`) for positioning: `'start' | 'center' | 'end'`
- Base UI Popover has no `Anchor` subcomponent — remove PopoverAnchor export if unused
- Search ALL consumers of a component when migrating (not just those listed in PRD notes) — e.g., QueryHistory.tsx also used PopoverTrigger asChild
