## Codebase Patterns
- electron-vite structure: src/main (node), src/preload (node), src/renderer (react)
- Dual vitest configs: vitest.config.main.ts (node env), vitest.config.renderer.ts (jsdom env)
- shadcn/ui components in src/renderer/src/components/ui/ - manually created, not CLI
- Tailwind CSS v3 with CSS variable-based theming (hsl values)
- Shared types in src/shared/types.ts - importable from both main and renderer
- Use `--passWithNoTests` flag with vitest run to allow zero test files
- Path alias: @renderer -> src/renderer/src (configured in electron.vite.config.ts and tsconfig.web.json)
- electron binary needs manual install.js run if pnpm ignored build scripts: `node node_modules/.pnpm/electron@28.3.3/node_modules/electron/install.js`
- React 18 + TypeScript, JSX transform via @vitejs/plugin-react
- EJSON import: use `import { EJSON } from 'mongodb/lib/bson'` (not directly exported from mongodb v6)
- EJSON Date serialization uses compact ISO format without trailing .000 (e.g. `2024-01-01T00:00:00Z`)
- MongoService pattern: all methods return `Result<T>`, catch errors internally
